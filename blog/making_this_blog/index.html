  • github: because I already have a github account

Setup

Those choices made, let’s setup the blog.
I am running an Ubuntu 18.04.4 LTS (lsb_release -a).

Hugo Install

# First install go
sudo snap install go --classic
# Then install hugo
sudo snap install hugo --channel=extended

Create blog site

I decided to simple name the website blog using the existing hugo-kiera theme:

# Create site
hugo new site blog
cd blog
# Install the theme
git init
cd themes
git submodule add https://github.com/funkydan2/hugo-kiera.git

To later update the theme, simply use:

cd themes
git submodule update --rebase --remote

Then edit the config.tom file at the blog root:

  • see the list of general parameters
  • see also the list of variables
  • see the Configuration of the hugo-keria theme
  • see the local file themes/hugo-kiera/exampleSite/config.toml

WARNING: when using github pages, it is important to add to the config.toml file

publishDir = "docs"

Its is the directory in which hugo will generate the static we site, which is also the repository expected by github to contains the web site.

Create a new article

If you are curious, you can learn more on the Hugo directory structure.

To add a new blog entry:

hugo new blog/my_new_post.md

One can also use static pages like an about page:

hugo new about.md

Use tags

Edit the newly created file:

vim content/blog/my_new_post.md

And add values in the tag and category header. E.g. :

tags: [rust, cow, parsing]
categories: [rust advanced]

Build and deploy

Build and test on localhost (including drafts)

hugo server -D

Build the site (including drafts)

hugo -D

Build (without draft)

hugo

Deploy and github (from the blog master directory)

git init
git add -A
git commit -m "First commit"
git remote add origin https://github.com/fxpineau/fxpineau.github.io.git
git push -u origin master
