Framework and Hosting

From a quick search, I decided to use

  • hugo: easy to set up and use, matches my simple needs, and fast
    • alternatives where e.g. Jekyll and possibly Gatsby
    • see for example this comparison
  • github: because I already have a github account and do no pretend to target a large audience

Setup

I am running an Ubuntu 18.04.4 LTS (lsb_release -a) so I am going to use the snap package manager. On MacOS, you can use brew.

Hugo Install

# First install go
sudo snap install go --classic
# Then install hugo
sudo snap install hugo --channel=extended

Create blog site

I decided to locally name my static website blog and to use the existing hugo-kiera theme:

# Create site
hugo new site blog
# Install the theme in the newly created site project
cd blog
git init
cd themes
git submodule add https://github.com/funkydan2/hugo-kiera.git

To later update the theme, simply use:

cd themes
git submodule update --rebase --remote

The next step is to edit the config.tom file at the blog root. To help you:

  • see the list of general parameters
  • see also the list of variables
  • see the Configuration of the hugo-keria theme
  • see the local file themes/hugo-kiera/exampleSite/config.toml

Here an example of config file used for this blog:

languageCode = "en-us"
theme = "hugo-kiera"

# Other params
canonifyURLs = true

paginate = 3

# Params given by hugo-keria
summaryLength = 30
enableEmoji = true
pygmentsCodeFences = true

Create a new article

To add a new blog entry:

hugo new posts/my_new_post.md

it will create the new markdown file in content/posts.

One can also use static pages like an about page:

hugo new about.md

If you are curious, you can learn more on the Hugo directory structure.

Edit a post and use tags and categories

Edit the newly created file:

vim content/blog/my_new_post.md

And fill the tag and category header fields. E.g. :

---
title: "Rusty Cow!"
date: 2020-05-15T18:00:00+02:00
tags: [rust, cow, parsing]
categories: [rust advanced]
---

See the list of possible emoji you can use in you markdow sources.

Create a github page

All information are available here, basically you only have to create a new github repository using the special name username.github.io (of course, you must replace username by your username).

Build and deploy

Generate the static files

Build and test on localhost (including drafts)

hugo server -D

Build the site (including drafts)

hugo -D

Build (without draft)

hugo

By default the site is created in the public directory, but you can change this default by setting the a variable in config.toml:

publishDir = "target/blog"

Deploy and github (poor man approach).

First, go into the hugo generated publishDir (public by default)

git init
git add -A
git commit -m "Commit of my blog"
git remote add origin https://github.com/username/username.github.io.git
git push -u origin master

WARNING: doing so, you keep the history of file generated by hugo which is not necessarily useful. So when generating a new version of your site, you may use

git add -A
git commit --amend
git push --force

A cleaner way is to setup a github action executing hugo on your versioned sources. To do so, see e.g.:

  • hugo action
  • or follow e.g. this tuto

Go check the result on https://username.github.io :)
